# SPAの動作確認

無難なSPAの模索。

あくまで実装例で、利用は非推奨です。

通常のアプリケーションでは、`laravel-app`, `rails-app`のような、MPA+island(Vue)までが無難。

## 構成

- Astro (SSG + ClientRouter) + island(Vue)
- NestJSのAPIサーバー

## 技術選定の理由

- Node SSRは、危なっかしいので、SSGを選択。（同じ理由で、Node SSRを使う場合はInertiaもおすすめできません。）
- Astro SSGのため、HTMLはページごとに分割され、JSもViteで最適化されて分割されるので、初期ロードが遅くなりにくい。
- SSGのため、SEO対策が、ある程度までは可能。
- 認証は安全性を優先してjwtトークンではなくセッションを選択。

## 機能

- 認証
- Twitter
- ユーザー管理画面

## 認証管理

### セッション

認証はセッションで管理。

### JS変数

`ClientRouter`を使っているので、ログイン情報は`Auth.user()`が最初にコールされたときにJS変数に保管することで、APIの送信を最小限にしている。

JS変数なのでリロード時にクリアされる。

ログイン、ログアウト時にもクリアしている。

SSGなので、モジュール内変数が利用可能なため、そこに保管している。(このような作りの為、SSRでは利用不可です。)

### その他

認証の判断は各API送信時にも行う。
