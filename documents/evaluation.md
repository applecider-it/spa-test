# 評価

Vueは、SEOが不要な場合は、JSがクライアントのみになり構造がシンプルになるので、比較的管理しやすい。

SEOが必要な場合は、AstroかNuxtJSになる。

どちらもprefetchできるので、パフォーマンスは大差なさそう。

管理画面のURLをわかりにくくしたいときは、Astro、NuxtJSはファイルベースルーティングのため、実装例のようになる。

NestJSをAPIにすることで、すべてJSで実装できるので、フルスタックエンジニアを見つけやすいかもしれない。

## Node SSRの危険性の説明

Node SSRは、アプリケーション内だけじゃなく、node_modulesの更新後のスレッドセーフの安全性の担保が、延々と必要になる。

もし、一つでも見落としがあると、ユーザー間の個人情報がSSRで出力される事故になる。

コミュニティで、ある程度、危険なモジュールがないか監視されているらしいけど、人力で１００％の安全性を延々と維持し続けなくてはいけなくなり、実用的ではないと思う。

特に、モジュール管理のスペシャリストがいない小規模な組織では、より一層、実用的ではないと思う。

node_modulesも完全にサーバーのみで動くものならいいけど、Node SSRで使うモジュールは、クライアントJSで動くことを想定して作られているため、Nodeで動かすと危険なことがある。
